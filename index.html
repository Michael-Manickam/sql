1. SQL Intro

Definition: Introduction to Structured Query Language used for managing relational databases.

Syntax: SELECT column_name FROM table_name;

2. SQL Select

Definition: Retrieves data from a database.

Syntax: SELECT column1, column2 FROM table_name;

3. SQL Select Distinct

Definition: Filters duplicate values from a result set.

Syntax: SELECT DISTINCT column_name FROM table_name;

4. SQL Where

Definition: Filters records based on specified conditions.

Syntax: SELECT column1, column2 FROM table_name WHERE condition;

5. SQL Order By

Definition: Sorts the result set in ascending or descending order.

Syntax: SELECT column1, column2 FROM table_name ORDER BY column_name ASC|DESC;

6. SQL And

Definition: Combines multiple conditions in a WHERE clause.

Syntax: SELECT column1, column2 FROM table_name WHERE condition1 AND condition2;

7. SQL Or

Definition: Retrieves records that meet either of multiple conditions.

Syntax: SELECT column1, column2 FROM table_name WHERE condition1 OR condition2;

8. SQL Not

Definition: Retrieves records that do not match a specified condition.

Syntax: SELECT column1, column2 FROM table_name WHERE NOT condition;

9. SQL Insert Into

Definition: Adds new records into a table.

Syntax: INSERT INTO table_name (column1, column2) VALUES (value1, value2);

10. SQL Null Values

Definition: Represents missing or unknown data in a database.

Syntax: SELECT column1, column2 FROM table_name WHERE column_name IS NULL;

11. SQL Update

Definition: Modifies existing records in a table.

Syntax: UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;

12. SQL Delete

Definition: Removes one or more records from a table.

Syntax: DELETE FROM table_name WHERE condition;

13. SQL Select Top

Definition: Retrieves a specific number of records from the top of a result set.

Syntax: SELECT TOP n column1, column2 FROM table_name;

14. SQL Aggregate Functions

Definition: Performs calculations on a set of values and returns a single result.

Syntax: SELECT AVG(column_name) FROM table_name;

15. SQL Min and Max

Definition: Returns the minimum or maximum value in a column.

Syntax: SELECT MIN(column_name) FROM table_name;

16. SQL Count

Definition: Counts the number of records that meet a specified condition.

Syntax: SELECT COUNT(column_name) FROM table_name WHERE condition;

17. SQL Sum

Definition: Calculates the sum of values in a column.

Syntax: SELECT SUM(column_name) FROM table_name;

18. SQL Avg

Definition: Calculates the average value of a numeric column.

Syntax: SELECT AVG(column_name) FROM table_name;

19. SQL Like

Definition: Retrieves records that match a specified pattern.

Syntax: SELECT column1, column2 FROM table_name WHERE column_name LIKE pattern;

20. SQL Wildcards

Definition: Special characters used with the LIKE operator to match patterns.

Syntax: SELECT column1, column2 FROM table_name WHERE column_name LIKE 'a%';

21. SQL In

Definition: Checks if a value matches any value in a list.

Syntax: SELECT column1, column2 FROM table_name WHERE column_name IN (value1, value2, ...);

22. SQL Between

Definition: Retrieves records within a range of values.

Syntax: SELECT column1, column2 FROM table_name WHERE column_name BETWEEN value1 AND value2;

23. SQL Aliases

Definition: Gives a table or column a temporary name for readability.

Syntax: SELECT column_name AS alias_name FROM table_name;

24. SQL Joins

Definition: Combines rows from two or more tables based on a related column.

Syntax: SELECT column1, column2 FROM table1 INNER JOIN table2 ON table1.column_name = table2.column_name;

25. SQL Inner Join

Definition: Returns only the rows with matching values in both tables.

Syntax: SELECT column1, column2 FROM table1 INNER JOIN table2 ON table1.column_name = table2.column_name;

26. SQL Left Join

Definition: Returns all rows from the left table and the matched rows from the right table.

Syntax: SELECT column1, column2 FROM table1 LEFT JOIN table2 ON table1.column_name = table2.column_name;

27. SQL Right Join

Definition: Returns all rows from the right table and the matched rows from the left table.

Syntax: SELECT column1, column2 FROM table1 RIGHT JOIN table2 ON table1.column_name = table2.column_name;

28. SQL Full Join

Definition: Returns all rows when there is a match in either left or right table.

Syntax: SELECT column1, column2 FROM table1 FULL JOIN table2 ON table1.column_name = table2.column_name;

29. SQL Self Join

Definition: Joins a table with itself to compare rows within the same table.

Syntax: SELECT a.column1, b.column2 FROM table_name a INNER JOIN table_name b ON a.common_column = b.common_column;

30. SQL Union

Definition: Combines the result sets of two or more SELECT statements.

Syntax: SELECT column1, column2 FROM table1 UNION SELECT column1, column2 FROM table2;

31. SQL Group By

Definition: Groups rows with the same values into summary rows.

Syntax: SELECT column1, COUNT(column2) FROM table_name GROUP BY column1;

32. SQL Having

Definition: Filters records based on group conditions.

Syntax: SELECT column1, COUNT(column2) FROM table_name GROUP BY column1 HAVING COUNT(column2) > value;

33. SQL Exists

Definition: Checks for the existence of rows in a subquery.

Syntax: SELECT column1 FROM table_name WHERE EXISTS (SELECT * FROM another_table WHERE condition);

34. SQL Any, All

Definition: Compares a value to a set of values using ANY or ALL operators.

Syntax: SELECT column1 FROM table_name WHERE column2 > ANY (SELECT column2 FROM another_table);

35. SQL Select Into

Definition: Copies data from one table into a new table.

Syntax: SELECT column1, column2 INTO new_table FROM old_table;

36. SQL Insert Into Select

Definition: Inserts data into a table from the result of a SELECT query.

Syntax: INSERT INTO table_name (column1, column2) SELECT column1, column2 FROM another_table WHERE condition;

37. SQL Case

Definition: Adds conditional logic to SQL queries.

Syntax:

        SELECT column_name,
            CASE
                WHEN condition1 THEN 'Result1'
                WHEN condition2 THEN 'Result2'
                ELSE 'Result3'
            END
        FROM table_name;

38. SQL Null Functions

Definition: Handles NULL values in SQL queries.

Syntax: SELECT column1, COALESCE(column2, default_value) FROM table_name;

39. SQL Stored Procedures

Definition: Precompiled SQL code that can be reused.

Syntax for creating:

        CREATE PROCEDURE procedure_name
        AS
        BEGIN
            SQL statements;
        END;

40. SQL Comments

Definition: Adds comments to SQL code for documentation or clarity.

Syntax:

Single line comment: -- Single line comment

Multi-line comment: /* Multi-line comment */

41. SQL Operators

Definition: Symbols used for arithmetic, comparison, and logical operations in SQL queries.

Syntax examples:

  • Arithmetic operators: +, -, *, /
  • Comparison operators: =, <>, <, >, <=, >=
  • Logical operators: AND, OR, NOT

42. SQL Create DB

Definition: Creates a new database in SQL.

Syntax: CREATE DATABASE database_name;

43. SQL Drop DB

Definition: Deletes an existing database in SQL.

Syntax: DROP DATABASE database_name;

44. SQL Backup DB

Definition: Creates a backup of a database.

Syntax: BACKUP DATABASE database_name TO disk = 'backup_location';

45. SQL Create Table

Definition: Creates a new table in a database.

Syntax:

CREATE TABLE table_name (
            column1 datatype,
            column2 datatype,
            ...
        );

46. SQL Drop Table

Definition: Deletes an existing table from a database.

Syntax: DROP TABLE table_name;

47. SQL Alter Table

Definition: Modifies an existing table structure.

Syntax examples:

  • Add a new column: ALTER TABLE table_name ADD column_name datatype;
  • Modify column: ALTER TABLE table_name ALTER COLUMN column_name datatype;
  • Drop column: ALTER TABLE table_name DROP COLUMN column_name;

48. SQL Constraints

Definition: Rules applied to columns to enforce data integrity.

Syntax examples:

  • Primary Key: CREATE TABLE table_name (column1 datatype PRIMARY KEY, ...);
  • Foreign Key: CREATE TABLE table_name (column1 datatype, column2 datatype, FOREIGN KEY (column1) REFERENCES other_table(column_name));
  • Not Null: CREATE TABLE table_name (column1 datatype NOT NULL, ...);

49. SQL Unique

Definition: Ensures that values in a column are unique.

Syntax: CREATE TABLE table_name (column1 datatype UNIQUE, ...);

50. SQL Primary Key

Definition: Uniquely identifies each record in a table.

Syntax: CREATE TABLE table_name (column1 datatype PRIMARY KEY, ...);

51. SQL Foreign Key

Definition: Establishes a link between two tables based on a key.

Syntax: CREATE TABLE table_name (column1 datatype, column2 datatype, FOREIGN KEY (column1) REFERENCES other_table(column_name));

52. SQL Check

Definition: Defines a condition that values must meet for a column.

Syntax: CREATE TABLE table_name (column1 datatype CHECK (condition), ...);

53. SQL Default

Definition: Sets a default value for a column when no value is specified.

Syntax: CREATE TABLE table_name (column1 datatype DEFAULT default_value, ...);

54. SQL Index

Definition: Improves the speed of data retrieval operations on a table.

Syntax: CREATE INDEX index_name ON table_name (column_name);

55. SQL Auto Increment

Definition: Automatically generates a unique value for a column.

Syntax (MySQL): CREATE TABLE table_name (column_name INT AUTO_INCREMENT PRIMARY KEY, ...);

Syntax (PostgreSQL): CREATE SEQUENCE sequence_name;

56. SQL Dates

Definition: Deals with date and time data types and functions.

Examples:

  • Current date: SELECT CURRENT_DATE;
  • Current time: SELECT CURRENT_TIME;
  • Date functions: SELECT DATE_ADD(date_column, INTERVAL 1 DAY) FROM table_name;

57. SQL Views

Definition: Virtual tables derived from one or more tables.

Syntax: CREATE VIEW view_name AS SELECT column1, column2 FROM table_name WHERE condition;

58. SQL Injection

Definition: Security vulnerability where malicious SQL statements are inserted into input fields.

Example (vulnerable code):

SELECT * FROM users WHERE username = 'input_username' AND password = 'input_password';

59. SQL Hosting

Definition: Services that provide SQL database hosting and management.

Example hosting providers:

  • Amazon RDS
  • Google Cloud SQL
  • Microsoft Azure SQL Database

60. SQL Data Types

Definition: Specifies the type of data that can be stored in a column.

Syntax examples:

  • Numeric data types: INT, DECIMAL, FLOAT
  • Character data types: CHAR, VARCHAR
  • Date and time data types: DATE, TIMESTAMP
